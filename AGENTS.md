# TensorTutors Agent Instructions

## Project identity

TensorTutors is a computer science learning platform.

The goal is to build a polished, static-first website for GCSE, A-Level, and early university Computer Science students.

The site should feel like an interactive textbook: clear, structured, visual, and learning-focused. Avoid blog-style waffle.

## Product goals

Build a website that helps students:
- Learn Computer Science topics quickly
- Practise exam-style questions
- Use interactive visualisers
- Follow clear learning roadmaps
- Revise with concise notes and quizzes

## Design direction

Use a dark academic tech aesthetic.

Style should be:
- clean
- modern
- sharp
- calm
- premium
- readable
- not childish
- not overloaded with animations

Use restrained motion:
- hover effects
- smooth transitions
- subtle card movement
- no excessive gimmicks

## Initial tech stack

Use:
- Vite
- React
- TypeScript
- Tailwind CSS

No backend yet.
No authentication yet.
No database yet.
No payments yet.
No deployment yet.

## Main site sections

The site should eventually contain:

- Projects

## Learning page structure

Lessons should generally follow:

1. Explain
2. Visualise
3. Practise
4. Test

A good lesson page includes:
- title
- short explanation
- learning objectives
- worked example
- visual aid or interaction
- common mistakes
- mini quiz
- next steps

## Coding rules

- Keep code modular.
- Prefer reusable components.
- Keep files organised.
- Use TypeScript types where useful.
- Avoid unnecessary dependencies.
- Do not add backend logic unless explicitly requested.
- Do not deploy unless explicitly requested.
- Do not change GitHub Pages settings unless explicitly requested.

## Branching rules

Use feature branches.
Do not work directly on main unless explicitly requested.

Recommended branches:
- feature/initial-setup
- feature/homepage
- feature/topic-system
- feature/visualisers
- feature/quiz-system

## Current first milestone

The first milestone is to create a clean static frontend MVP with:
- React + TypeScript + Tailwind
- responsive navigation
- homepage
- basic route structure
- placeholder pages
- reusable cards/components
- local data files for topics
